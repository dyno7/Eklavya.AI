# Plan 10.1: Backend XP Engine — Multipliers, Bonuses & Badge Auto-Award

<objective>
Enhance `claim_task` to produce richer XP events (streak multiplier, milestone/goal bonuses, level-up detection) and auto-award badges when thresholds are met. No new DB tables needed — badge seeding only.

Purpose: Deepen the XP/gamification loop on the backend.
Output: Updated `claim_task` endpoint returning rich XP data.
</objective>

<task type="auto">
  <name>Enhance claim_task with streak multiplier, milestone/goal bonuses, level-up detection</name>
  <files>
    eklavya_backend/app/presentation/dashboard.py
  </files>
  <action>
    1. Update `ClaimTaskResponse` schema to add: `bonus_xp: int`, `level_up: bool`, `new_level: int`, `badges_awarded: list[str]`.
    2. Before awarding XP, capture `old_xp = user.total_xp or 0` and `old_level = old_xp // 100`.
    3. Compute `streak = user.current_streak or 0`. Set multiplier: `streak >= 7 → 1.5`, `streak >= 3 → 1.2`, else `1.0`.
    4. `bonus_xp = int(task.xp_reward * (multiplier - 1.0))`. Already-existing code awards `task.xp_reward`; extend to award `task.xp_reward + bonus_xp`.
    5. After milestone completion block, award +50 XP bonus. After goal completion block, award +200 XP bonus.
    6. Compute `new_level = user.total_xp // 100`. Set `level_up = new_level > old_level`.
    7. In return statement, populate the new fields.
    AVOID: Breaking existing milestone/goal advancement logic because the frontend relies on it. Do NOT change the streak update logic.
  </action>
  <verify>curl -s -X POST http://127.0.0.1:8000/api/v1/dashboard/claim-task/{TASK_ID} -H "Authorization: Bearer TOKEN" | python -m json.tool | grep -E "bonus_xp|level_up|new_level"</verify>
  <done>Response JSON includes `bonus_xp` (int), `level_up` (bool), `new_level` (int), `badges_awarded` (list)</done>
</task>

       Query `Badge` by name, check `UserBadge` existence. If missing, insert `UserBadge` and return badge name. Use `async def`.
    2. In `claim_task`, after XP commit, evaluate badge triggers:
       - Always check "First Steps" (first task completion)
       - `total_xp >= 100` → "Novice"
       - `total_xp >= 500` → "Centurion"
       - `current_streak >= 7` → "Consistency"
       - If milestone just completed → "Milestone Master"
       - If goal just completed → "Goal Crusher"
    3. Collect non-None results in `badges_awarded: list[str]` and include in response.
    AVOID: Querying badges table on every claim if user has no triggers close to thresholds — use simple checks first.
  </action>
  <verify>Complete the very first task for a fresh user → response.badges_awarded should include "First Steps"</verify>
