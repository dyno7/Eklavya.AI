# Phase 11.1: Data Layer for Goal Drift Index

## Objective
Establish strict relational tables needed to compute the complex `GDI(t)` equation, track app opens without completions (V), and log episodic rewards.

## Tasks
1. **Create `UserSessionLog` Model**
   - Tracks app opens to calculate `V(t)` (opens without tasks completed).
   - Fields: `id`, `user_id`, `login_timestamp`, `tasks_completed_in_session` (Int).

2. **Create `UserBehaviorLog` Model**
   - Aggregates daily GDI elements (State Estimator).
   - Fields: `id`, `user_id`, `date`, `momentum_score` (Float), `avoidance_count` (Int: easy tasks done when hard skipped), `decay_value` (Float), `gdi_score` (Float).

3. **Create `RewardSignalLog` Model**
   - Stores episodic vectors.
   - Fields: `id`, `user_id`, `timestamp`, `action_type`, `reward_value`.

4. **Schema Migrations**
   - Run `alembic revision --autogenerate -m "Add GDI and Session telemetry"`.
   - Apply migration to the PostgreSQL DB.
