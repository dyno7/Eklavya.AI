# Plan 9.1: Milestone Progress Sync & Live UI Refresh

## Objective
Make task completion feel real-time end-to-end. When a user completes a task, milestone progress, goal status, XP, and streak all update correctly in the backend — and every affected Flutter tab (Home, Goals, Profile, Analytics) refreshes automatically without manual navigation.

## Context
- eklavya_backend/app/presentation/dashboard.py
- eklavya_backend/app/core/repositories.py
- eklavya_backend/app/domain/enums.py
- eklavya_mobile/lib/core/services/dashboard_service.dart
- eklavya_mobile/lib/features/dashboard/home_tab.dart
- eklavya_mobile/lib/features/goals/goals_tab.dart
- eklavya_mobile/lib/features/goals/goal_roadmap_screen.dart
- eklavya_mobile/lib/features/analytics/analytics_tab.dart

## Tasks

<task type="auto">
  <name>Backend: Milestone Auto-Advance on Task Completion</name>
  <files>eklavya_backend/app/presentation/dashboard.py, eklavya_backend/app/core/repositories.py</files>
  <action>
    In `claim_task` endpoint (dashboard.py lines 161–226), after completing a task:
    1. Reload ALL tasks for the milestone (not cached) to get fresh `TaskStatus`.
    2. If ALL tasks in the milestone are COMPLETED → set milestone status to COMPLETED.
    3. Find the next milestone (lowest order_index that is LOCKED) → set it to ACTIVE.
    4. If ALL milestones for the goal are COMPLETED → set goal status to COMPLETED.
    This logic already exists partially (lines 183–201) — verify it fires correctly for the current task's milestone by using `task.milestone_id`. Bug: the check `all(t.status == TaskStatus.COMPLETED for t in milestone_tasks)` may miss the just-completed task because `update_task_status` was called but the cached `milestone_tasks` list still shows old status. Fix: reload tasks AFTER `update_task_status` returns. Do NOT change to a different ORM pattern.
  </action>
  <verify>In the Supabase dashboard, complete all tasks in a milestone and verify the milestone row's status changes to 'completed' and the next milestone changes from 'locked' to 'active'.</verify>
  <done>POST /api/v1/dashboard/claim-task/{task_id} auto-advances milestone and activates the next one when all tasks are done. GET /api/v1/dashboard/summary returns updated completed_milestones count.</done>
</task>

  <files>eklavya_mobile/lib/core/services/dashboard_service.dart, eklavya_mobile/lib/features/goals/goal_roadmap_screen.dart, eklavya_mobile/lib/features/dashboard/home_tab.dart, eklavya_mobile/lib/features/analytics/analytics_tab.dart, eklavya_mobile/lib/features/profile/profile_tab.dart</files>

    3. In `home_tab.dart`, `analytics_tab.dart`, and `profile_tab.dart` — if they use `FutureProvider` or `StateNotifier`, ensure they listen to the same provider so invalidation cascades (or call `.refresh()` on mount via `ref.listen`).

## Success Criteria
- [ ] Completing all tasks in a milestone auto-completes the milestone and activates the next.
- [ ] Goal status updates to COMPLETED when all milestones are done.
- [ ] Home, Goals, Analytics, Profile tabs refresh automatically after task completion.
- [ ] No polling timers — refresh is event-driven only.
