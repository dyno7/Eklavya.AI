# Plan 9.2: Coach Agent — RL-Based Drift Detection & Plan Auto-Adjustment

## Objective
Build the passive background intelligence layer of Eklavya. A `CoachAgent` uses reinforcement-learning-inspired drift signals (inactivity, skipped tasks, streak breaks) to detect when a user is falling behind and emit coach interventions. A new backend endpoint exposes the coach status, and the Flutter app surfaces nudges non-intrusively on the Home tab.

  <files>eklavya_backend/app/agents/coach_agent.py (NEW), eklavya_backend/app/presentation/coach.py (NEW), eklavya_backend/app/main.py</files>
  <action>
    Create `eklavya_backend/app/agents/coach_agent.py`:
    ```python
    class CoachAgent:
        """RL-inspired drift detector. Stateless — evaluates signals on each call."""

        DRIFT_THRESHOLDS = {
            "inactivity_days": 3,      # No task completed in N days
            "skip_rate": 0.4,          # >40% of recent tasks skipped
            "streak_broken": True,     # Streak dropped to 0 after >2
        }

        def evaluate(self, user_stats: dict, recent_tasks: list[dict]) -> dict:
            """
            Returns: {
              "drift_detected": bool,
              "drift_type": str | None,  # "inactivity" | "skip_rate" | "streak_break"
              "severity": str,           # "low" | "medium" | "high"
              "nudge": str,              # Plain-language coaching message
              "adjusted_daily_tasks": int | None  # Suggested load reduction (1–3)
            }
            """
    ```
    Drift logic:
    - **Inactivity**: `last_activity` is None or > 3 days ago → severity "high"
    - **Skip rate**: count tasks with status=="skipped" in last 10 tasks / total > 0.4 → severity "medium"
    - **Streak break**: `current_streak == 0` AND user has ever had streak > 2 → severity "low"
    Generate `nudge` as a short motivational string (no Gemini API call — use rule-based templates per drift_type).
    `adjusted_daily_tasks` = max(1, current_pending_count - 1) if medium/high severity.

    Create `eklavya_backend/app/presentation/coach.py`:
    - `GET /api/v1/coach/status` — requires JWT auth
    - Reads user stats (XP, streak, last_activity from profile), last 10 tasks from repositories
    - Calls `CoachAgent().evaluate(...)` 
    - Returns the evaluate dict + current streak + days_since_activity

    Wire it in `main.py`: `app.include_router(coach_router)`.
    Do NOT add any database migration — coach uses existing tables (user_profiles, tasks) read-only.
  </action>

    curl -H "Authorization: Bearer {JWT}" http://localhost:8000/api/v1/coach/status
    Expected: JSON with drift_detected, nudge, severity fields.
  </verify>
  <done>GET /api/v1/coach/status returns a valid JSON coach evaluation. Returns 200 for authenticated user. Drift detected=true if user has 3+ day gap in task completions.</done>
