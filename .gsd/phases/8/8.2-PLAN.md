# Plan 8.2: Chatbot Memory & Roadmap Context

## Objective
Fix the fundamental issue where the Guru Agent loses conversation history between messages and is completely unaware of the user's active roadmap/goals.

## Context
- eklavya_backend/app/agents/guru_agent.py
- eklavya_backend/app/presentation/chat.py

## Tasks

<task type="auto">
  <name>Dynamic Roadmap Context Injection</name>
  <files>eklavya_backend/app/presentation/chat.py, eklavya_backend/app/agents/guru_agent.py</files>
  <action>
    Fetch the user's active goals and tasks and pass them into the agent.
    - In `chat.py`, use the DB connection to query the user's active `goals` where `status='active'`.
    - Modify the `GuruAgent.__init__` signature to optionally accept `roadmap_context: str` and append that context dynamically to the `system_instruction`.
  </action>
  <verify>grep "roadmap_context" eklavya_backend/app/agents/guru_agent.py</verify>
  <done>System instruction explicitly informs the agent of the user's current progress</done>
</task>

<task type="auto">
  <name>Conversation History & Navigation Signal</name>
  <files>eklavya_backend/app/agents/guru_agent.py, eklavya_backend/app/presentation/chat.py</files>
  <action>
    Prevent the agent from resetting memory on every API call.
    - The `_sessions[user_id]` dict must preserve the active `model.start_chat(history=...)` session across API requests so that the `_chat.send_message()` call has historical context.
    - Instruct the Agent inside the system prompt: "If the user asks to see their roadmap or asks to start, reply using a strict JSON block `{\"navigate_to\": \"roadmap\", \"message\": \"...\"}`".
    - Update `ChatSendResponse` pydantic model in `chat.py` to parse this structure safely and provide a `navigate_to_roadmap=True` boolean back to the Flutter UI.
  </action>
  <verify>uv run pytest (or manual verification if no tests)</verify>
  <done>Bot remembers previous message in a follow-up hit and returns the navigate flag correctly</done>
</task>

## Success Criteria
- [ ] Agent correctly refers back to goals previously generated when asked about their progress.
- [ ] Calling the chat endpoint with "Take me to my roadmap" returns `navigate_to_roadmap=True`.
