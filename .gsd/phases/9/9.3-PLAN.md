# Plan 9.3: Persistent Chat Memory & Final E2E

## Objective
Wire the existing `chat_messages` DB table to persist every Guru conversation turn. On the next session, the last 10 messages are injected into the Guru as `memory_context` so the user can ask "what was my roadmap?" and get a real answer. Final E2E smoke test + STATE/ROADMAP update.

  <action>
    In `repositories.py` add:
    ```python
    async def save_chat_message(db: AsyncSession, user_id: str, role: str, content: str) -> None:
        """Persist a single chat turn to chat_messages table."""
        msg = ChatMessage(user_id=user_id, role=role, content=content)
        db.add(msg)
        await db.commit()

    async def get_recent_chat_messages(db: AsyncSession, user_id: str, limit: int = 10) -> list[dict]:
        """Return last N messages ordered chronologically (oldest first)."""
        result = await db.execute(
            select(ChatMessage)
            .where(ChatMessage.user_id == user_id)
            .order_by(ChatMessage.created_at.desc())
            .limit(limit)
        )
        msgs = result.scalars().all()
        return [{"role": m.role, "content": m.content} for m in reversed(msgs)]
    ```
    Import `ChatMessage` from `app.domain.models` (already defined from Phase 1 schema).

    In `chat.py` POST `/api/v1/chat/message`:
    1. BEFORE constructing `GuruAgent`: call `get_recent_chat_messages(db, user_id)`.
       Format as: `"Turn 1 — User: {content}\nTurn 1 — Guru: {content}\n..."`.
       Pass as `memory_context=` kwarg to `GuruAgent(...)`.
    2. AFTER getting the reply tuple `(reply, is_ready, navigate)`:
       Call `await save_chat_message(db, user_id, "user", user_message)`.
       Call `await save_chat_message(db, user_id, "assistant", reply)`.
    
    Constraint: Do NOT change the `chat_messages` table schema. Use only existing columns.
    Constraint: Do NOT call `save_chat_message` if the reply was a demo/offline response.
