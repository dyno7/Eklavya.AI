# Phase 11.2: GDI & State Estimation Mathematical Engine

## Objective
Translate the mathematical GDI formula `GDI(t) = a.M(t) + b.V(t) + g.c(t) + d.D(t)` and "Commitment Gradient Descent" into actionable backend Python services.

## Tasks
1. **Develop `GdiService`**
   - Create a service class calculating variables on the fly:
     - `M(t)`: Exponentiated moving average of XP velocities.
     - `V(t)`: Queries `UserSessionLog` to find open intervals where 0 tasks were claimed.
     - `c(t)`: Queries completed tasks checking if lower difficulty tasks were picked over available chronological hard ones.
     - `D(t)`: Task decay tracker (scaling time since last completion).
   - Evaluate `GDI(t)` score using pre-defined weights (`alpha`, `beta`, `gamma`, `delta`).

2. **Commitment Gradient Descent Interventions**
   - If `GDI(t)` falls past a negative threshold, output graceful intervention states instead of spammy notifications.
   - States: `ENGAGED`, `WAVERING`, `SILENT_RECESS`.

3. **Background Decay Sweeper (APScheduler)**
   - Setup a lightweight APScheduler Cron triggering at Midnight.
   - For all active users, step `D(t)` forward, update `UserBehaviorLog`, and persist the history for the State Estimator.
